# AI 标书写作平台开发周期与报价估算

## 估算结论

按当前 MVP 范围估算，推荐先做 **MVP 可用版**。

| 版本 | 开发周期 | 报价区间 | 说明 |
| --- | --- | --- | --- |
| 演示版 | 3-5 周 | 8-15 万 | 能跑通流程，适合演示，不适合正式商用 |
| MVP 可用版 | 8-12 周 | 25-45 万 | 推荐，支持试点客户使用 |
| 商业上线版 | 12-16 周 | 40-70 万 | 稳定性、安全、后台、异常处理更完整 |
| 完整功能版 | 5-8 个月 | 80-150 万+ | 覆盖原 Word 里的 44 个功能，不建议第一阶段做 |

建议第一阶段按 **MVP 可用版** 做，预算控制在 **30-35 万** 比较合理。

## MVP 核心范围

- 上传招标文件。
- PDF 文本解析。
- AI 解读评分点、资质要求、废标条款。
- 自动生成标书目录。
- 分章节 AI 写作。
- 基础在线编辑。
- Word 导出。
- 废标风险检查。
- Token 用量记录，但不接真实支付。

## 额外功能增量估算

| 功能 | 增加周期 | 增加费用 |
| --- | --- | --- |
| 扫描件 OCR | +2-4 周 | +3-8 万 |
| 微信/支付宝支付 | +2-3 周 | +3-6 万 |
| 企业知识库 | +4-6 周 | +6-12 万 |
| 多人协作 | +4-8 周 | +8-15 万 |
| PPT 自动生成 | +3-5 周 | +5-10 万 |

## 估算前提

- MVP 优先支持文字型 PDF，扫描件 OCR 不作为第一阶段必做。
- 不接微信/支付宝真实支付。
- 不做完整知识库、多团队协作、PPT、AI 生图。
- Word 导出只做 1-2 套固定模板。
- 使用现成大模型 API，不自研模型。
