# 新增需求归档规则

后续新来的需求按“功能相似度”归到一起，不按日期散放。

原则：

1. **先找相同模块**：同类需求放进同一个模块目录。
2. **再看是否同一个主题**：如果是同一个主题的补充，追加到已有需求文件，不要重复新建。
3. **无法判断时先放待分类**：后续评估后再移动到正确模块。
4. **评估确认后再同步 PRD**：本目录是需求池，不直接代表已经进入 MVP。

## 分组目录

| 目录 | 放什么需求 |
| --- | --- |
| `01-文件解析与OCR/` | PDF 上传、PDF 文本解析、扫描件 OCR、文件格式支持 |
| `02-AI解读与写作/` | 招标文件解读、评分点提取、目录生成、章节写作、提示词优化 |
| `03-编辑器与Word导出/` | 在线编辑器、Word 模板、目录页码、排版、PDF 导出 |
| `04-风险检查与质量保障/` | 废标检查、评分自检、查重、内容质量检查 |
| `05-账号会员计费后台/` | 登录注册、会员、Token 计费、支付、管理后台 |
| `06-模板素材知识库/` | 标书模板、素材库、企业知识库、历史标书库 |
| `07-协作权限与团队/` | 多人协作、团队、角色权限、评论批注 |
| `08-部署安全与运维/` | 部署、服务器、数据安全、日志、监控、备份 |
| `99-待分类/` | 暂时无法判断归属的需求 |

## 文件命名

模块目录内建议使用：

```text
YYYY-MM-DD-需求名称.md
```

如果后续来的内容和已有需求是同一主题，直接追加到原文件的“补充记录”里。

示例：

```text
01-文件解析与OCR/2026-06-18-扫描件OCR支持.md
05-账号会员计费后台/2026-06-18-微信支付接入.md
```
