# 新增需求归档规则

后续新来的需求先放在本目录，评估确认后再同步到对应 PRD 或报价文档。

## 文件命名

建议使用：

```text
YYYY-MM-DD-需求名称.md
```

示例：

```text
2026-06-18-扫描件OCR支持.md
2026-06-18-微信支付接入.md
```

## 单个需求建议格式

```markdown
# 需求名称

## 背景

说明为什么需要这个需求。

## 需求内容

- 功能点 1
- 功能点 2

## 优先级

P0 / P1 / P2

## 是否进入 MVP

是 / 否 / 待评估

## 影响范围

- 产品
- 前端
- 后端
- AI
- Word 导出
- 部署/运维

## 周期与成本影响

待评估。
```
